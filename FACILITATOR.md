# FACILITATOR.md — 幫別人建 Agent 的指南

> 這份文件是給「扮演顧問角色、幫別人安裝和設定 AI Agent 系統」的人看的。

---

## 核心理念

不是「教課」，是「帶人玩」。

邊裝邊解釋，讓使用者自己按按鈕。他們不需要懂所有原理，他們需要的是第一次「哇！」的體驗——AI 認識自己、幫自己做了一件事。

**你的角色：** 降低阻力，製造成就感，讓他們在 2 小時內感覺「我做到了」。

---

## 教學金句庫（可直接用的類比）

當使用者聽到陌生術語時，這些類比能瞬間讓他們理解：

- **API：** 「API 就是電腦跟電腦講話的方式。就像你打電話給客服，有固定的流程——API 就是那個流程。」
- **GitHub：** 「GitHub 像雲端硬碟，但免費、有版本控制。每次改動都有記錄，永遠可以回頭看。」
- **為什麼要這套系統：** 「你可以想像你就跟你的電腦自言自語，然後它還幫你梳理很多脈絡。」
- **備份安全感：** 「不管重裝 OpenClaw 或換電腦，你的知識都會在，有一個大腦在這邊。」
- **Agent 獨立運作：** 「就像放出去的小孩——你不用一直盯著，它自己會去做事，做完回來報告。」

## 多線並行技巧

2 小時很短。學會同時推進兩件事：

| 等待時機 | 同步去做 |
|----------|----------|
| `npm install` 安裝等待 | → 幫使用者設 Discord webhook |
| AI 跑訪談問答 | → 設定 GitHub repo 備份 |
| OAuth 驗證等瀏覽器跳轉 | → 解釋系統架構給使用者聽 |
| Google API 啟用審核等待 | → 先測試其他功能 |

**原則：永遠有兩件事同時在推進。** 死等 = 浪費雙方的時間。

## AI 主導訪談（殺手功能）

> 這是整個系統最讓人驚豔的環節。不要跳過。

### 為什麼不用手動填表格

手動填 USER.md 太枯燥，使用者不知道寫什麼，顧問也要一直問問題很累。

**更好的方式：** 讓 AI 自己問。

### 訪談流程

1. **先讓 Agent 自我介紹**（讓使用者感受一下這個「新生命」）
2. **Agent 問開放式問題：**
   - 你做什麼工作？最近在忙什麼？
   - 你目前最頭痛的事情是什麼？
   - 你平常用什麼工具？什麼讓你最崩潰？
   - 你的作息是怎樣的？幾點最有精神？
   - 你有什麼長期想做但一直沒做的事？
   - 你希望這個 AI 夥伴能幫你什麼？
3. **使用者用 Whisper Flow（或其他語音輸入工具）口述回答**
   - 語音比打字自然 10 倍
   - 讓對話流動，不要催
   - 20 分鐘的人生故事，比填表格深很多
4. **Agent 自動整理成 MEMORY.md + USER.md**
5. **顧問這時候在旁邊做其他設定**（多線並行！）

### 語音輸入前置設定

```
系統偏好設定 → 隱私與安全 → 麥克風 → 允許對應 App
Whisper Flow 需要先授權麥克風才能使用
```

## 安全教育 Checklist（現場教，不跳過）

這些不是嚇人的，是真的會遇到的情況：

- [ ] 示範：Google 搜尋第一個結果可能是釣魚廣告（跟使用者一起辨識廣告標記）
- [ ] **API key = 密碼**，不要截圖、不要傳 LINE/Messenger/社群
- [ ] 權限最小化原則：需要什麼就開什麼，不要一次全開
- [ ] GitHub repo 記得設 **Private**（除非刻意要公開）
- [ ] `client_secret.json` 不要放在桌面或容易截圖到的地方

## 時間管理

**嚴格 2 小時。** 這是職業標準，友情價也不例外。

- 開場就設手機計時器，讓使用者看到
- 最後 10 分鐘 = 帶走包生成（TAKEAWAY.md）+ Q&A
- 到時間就結束，就算沒裝完也結束
  - 未完成的部分給他們作業，或排第二次
- 超時 = 不專業，會讓使用者覺得你對時間沒把握

### 話題發散時的收尾語

> 「這個很棒，先記下來，我們先把核心搞定，之後可以再深入。」

（同時在 AI 對話裡留一個待辦：「記下來：使用者想深入了解 XX」）

## 常見踩坑 & 解法

| 坑 | 解法 |
|----|------|
| Google OAuth 設定卡住 | 事先截圖每一步，準備好說明圖。這是最常卡的環節。 |
| 使用者對 Git 完全陌生 | 用「雲端備份有版本記錄」類比，不要深講 Git 概念 |
| Obsidian 初次開啟需要 iCloud 同步 | 提前告知，讓他們早點打開，趁安裝其他東西時等 |
| Whisper Flow 忘記授權麥克風 | 開始前就先處理，不要等要用才發現 |
| 使用者問「這跟 ChatGPT 有什麼不同」 | 「ChatGPT 每次都是陌生人，這個是認識你的朋友。」 |
| 2 小時裝不完 | 正常的。最後 10 分鐘做帶走包，剩下的變作業 |
| 使用者突然想聊深度問題 | 「先把基礎裝好，有了系統我們之後可以繼續聊。」 |

## 帶走包生成（每場必做）

諮詢結束前 10 分鐘，讓 AI 產出 TAKEAWAY.md：

```
請幫我產生今天的帶走包，記錄：
1. 今天設定好了什麼（勾選清單）
2. 明天的第一個動作
3. 這週的 3 件功課
格式用 TAKEAWAY.md 模板。
```

讓使用者讀一遍、確認，然後截圖或存下來。
